# Additional Learning Resources

Congratulations on completing (or working through) the Introduction to GitHub course! Here are additional resources to continue your learning journey.

## GitHub Learning Resources

### GitHub Skills Courses
Continue your learning with more interactive GitHub Skills courses:
- [Communicate using Markdown](https://github.com/skills/communicate-using-markdown) - Learn to use Markdown to add formatting to your writing
- [Review pull requests](https://github.com/skills/review-pull-requests) - Learn how to review pull requests effectively
- [Resolve merge conflicts](https://github.com/skills/resolve-merge-conflicts) - Learn how to resolve merge conflicts
- [Release-based workflow](https://github.com/skills/release-based-workflow) - Learn about release workflows with GitHub
- [Connect the dots](https://github.com/skills/connect-the-dots) - Learn about GitHub Actions and continuous integration

### GitHub Documentation
- [GitHub Docs](https://docs.github.com) - Comprehensive documentation for all GitHub features
- [Git Handbook](https://guides.github.com/introduction/git-handbook/) - Learn the basics of Git
- [GitHub Guides](https://guides.github.com) - Short guides on GitHub topics
- [GitHub Learning Lab](https://lab.github.com) - Interactive courses and tutorials

## Git Resources

### Learning Git
- [Pro Git Book](https://git-scm.com/book/en/v2) - Free, comprehensive book about Git (available in multiple languages)
- [Git Cheat Sheet](https://education.github.com/git-cheat-sheet-education.pdf) - Quick reference for common Git commands
- [Learn Git Branching](https://learngitbranching.js.org) - Interactive visual tutorial for Git branching
- [Oh My Git!](https://ohmygit.org) - An open-source game about learning Git

### Git Command Reference
- [Git Reference](https://git-scm.com/docs) - Official Git command reference
- [Git Tips](https://github.com/git-tips/tips) - Collection of useful Git tips

## Development Tools

### Code Editors
- [Visual Studio Code](https://code.visualstudio.com) - Popular free code editor with excellent Git integration
- [GitHub Codespaces](https://github.com/features/codespaces) - Cloud-based development environment
- [GitHub Desktop](https://desktop.github.com) - GUI application for managing Git repositories

### Browser Extensions
- [Refined GitHub](https://github.com/refined-github/refined-github) - Browser extension that enhances the GitHub interface
- [Octotree](https://www.octotree.io) - Browser extension for easy repository navigation

## Community and Collaboration

### Getting Involved
- [GitHub Explore](https://github.com/explore) - Discover interesting projects and topics
- [First Contributions](https://github.com/firstcontributions/first-contributions) - Help beginners make their first contribution
- [Good First Issues](https://goodfirstissue.dev) - Find projects with good first issues for beginners
- [Up For Grabs](https://up-for-grabs.net) - Projects with tasks specifically for new contributors

### GitHub Community
- [GitHub Community Forum](https://github.community) - Get help and connect with other GitHub users
- [GitHub Skills Discussions](https://github.com/orgs/skills/discussions) - Discuss GitHub Skills courses
- [GitHub Blog](https://github.blog) - Stay updated with GitHub news and features

## Best Practices

### Writing Great Documentation
- [Awesome README](https://github.com/matiassingers/awesome-readme) - Examples of great README files
- [Make a README](https://www.makeareadme.com) - Guide to writing good READMEs
- [Markdown Guide](https://www.markdownguide.org) - Complete guide to Markdown

### Collaboration Best Practices
- [How to Write a Good Commit Message](https://cbea.ms/git-commit/) - Guide to writing clear commit messages
- [How to Write the Perfect Pull Request](https://github.blog/2015-01-21-how-to-write-the-perfect-pull-request/) - Best practices for PRs
- [Code Review Best Practices](https://github.com/features/code-review/) - Tips for effective code reviews

### Project Management
- [GitHub Issues](https://docs.github.com/en/issues) - Learn to use issues effectively
- [GitHub Projects](https://docs.github.com/en/issues/planning-and-tracking-with-projects) - Project management on GitHub
- [Agile Development on GitHub](https://resources.github.com/devops/process/agile/) - Implementing agile workflows

## Practice Projects

### Project Ideas
- Create a personal portfolio website using GitHub Pages
- Contribute to an open-source project
- Build a project and document it with a great README

The best way to learn is by doing. Don't be afraid to:
- Experiment with Git and GitHub features
- Make mistakes (you can always start over!)
- Ask questions in community forums
- Share your knowledge with others
- Contribute to open-source projects
