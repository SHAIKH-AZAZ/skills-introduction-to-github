# Contributing to Introduction to GitHub

Thank you for your interest in contributing to this GitHub Skills course! This document provides guidelines for contributing to this educational repository.

## How to Contribute

### Reporting Issues

If you find any issues with the course content, instructions, or workflows:

1. Check if the issue already exists in the [Issues](../../issues) section
2. If not, create a new issue with a clear description
3. Include steps to reproduce if applicable
4. Add relevant labels (e.g., `bug`, `documentation`, `enhancement`)

### Suggesting Improvements

We welcome suggestions for improving this learning experience:

- **Content Improvements**: Clarifications, better explanations, or additional examples
- **Technical Updates**: Fixing broken links, updating outdated information
- **New Features**: Additional learning steps or exercises

### Making Changes

1. **Fork the repository** to your own GitHub account
2. **Create a new branch** from `main` with a descriptive name (e.g., `fix-typo-readme`, `add-example-step2`)
3. **Make your changes** following the guidelines below
4. **Test your changes** by going through the course steps
5. **Submit a pull request** with a clear description of your changes

## Guidelines

### Documentation

- Use clear, concise language appropriate for beginners
- Include examples where helpful
- Follow the existing Markdown formatting style
- Check spelling and grammar

### Workflows

- Test any workflow changes thoroughly
- Ensure backward compatibility
- Document any new environment variables or secrets needed

### Images

- Use descriptive file names (e.g., `create-branch-button.png`)
- Keep images current with GitHub's UI
- Optimize images for web (reasonable file sizes)

## Code of Conduct

Please note that this project follows a [Code of Conduct](CODE_OF_CONDUCT.md). By participating, you are expected to uphold this code.

## Questions?

If you have questions about contributing, feel free to:

- [Open a discussion](https://github.com/orgs/skills/discussions/categories/introduction-to-github)
- Check the [GitHub Docs](https://docs.github.com)
